# Brain Gym — Development Guidelines

## Code Quality Standards

### TypeScript Conventions
- Strict TypeScript throughout both frontend and backend
- Interfaces preferred over types for object shapes (e.g., `interface Choice`, `interface Question`)
- Enums used for fixed value sets: `QuestionType`, `Difficulty`, `UserRole`, `QuestionStatus`
- Path aliases: `@/` maps to `src/` in frontend (configured in vite/tsconfig)
- Optional fields marked with `?` suffix consistently

### Naming Conventions
- **Files**: PascalCase for React components (`ExamPage.tsx`, `CertificationCard.tsx`), camelCase for services/utils (`auth.service.ts`, `mockData.ts`)
- **Components**: PascalCase function names, default export for page components, named exports for reusable components
- **Variables/Functions**: camelCase (`handleSubmit`, `selectAnswer`, `toggleMark`)
- **Constants**: camelCase for component-scoped, UPPER_SNAKE_CASE for module-level constants (`SIDEBAR_WIDTH`, `SIDEBAR_COOKIE_NAME`)
- **CSS classes**: Tailwind utility-first with custom classes (`glass-card`, `glow-cyan`, `text-gradient-cyan`, `bg-grid`)
- **Backend**: NestJS conventions — PascalCase for classes/decorators, camelCase for methods, `*.module.ts` / `*.service.ts` / `*.controller.ts` / `*.dto.ts` pattern

### Prisma Schema Conventions
- Model names: PascalCase singular (`User`, `Question`, `ExamAttempt`)
- Table names: snake_case plural via `@@map("table_name")`
- Column mapping: camelCase in code → snake_case in DB via `@map("column_name")`
- UUIDs as primary keys: `@id @default(uuid())`
- Timestamps: `createdAt DateTime @default(now())`, `updatedAt DateTime @updatedAt`
- Soft enums defined at schema level, referenced in models

## Architectural Patterns

### Frontend Component Structure
Pages are self-contained with inline state management, no separate container/presenter split:
```tsx
export default function PageName() {
  const navigate = useNavigate();
  const [state, setState] = useState(initialValue);
  // ... all logic inline
  return (<div className="min-h-screen bg-background">...</div>);
}
```

### State Management Pattern
- **Server state**: TanStack React Query with `useQuery` for data fetching
```tsx
const { data, isLoading, error } = useQuery({
  queryKey: ['certifications'],
  queryFn: getCertifications,
});
```
- **Client state**: Zustand stores with `persist` middleware for auth
```tsx
export const useAuthStore = create<AuthState>()(
  persist((set) => ({ /* state + actions */ }), { name: 'auth-storage' })
);
```
- **Local state**: React `useState` for form/UI state within components

### API Service Layer Pattern
All API calls go through a centralized Axios instance with interceptors:
```tsx
// services/api.ts — base instance with auth interceptor + token refresh
// services/questions.ts — domain-specific functions
export const getQuestions = async (certificationId?: string, page = 1, limit = 10): Promise<PaginatedQuestions> => {
  const response = await api.get<PaginatedQuestions>(`/questions?${params.toString()}`);
  return response.data;
};
```
- Services return unwrapped `response.data`, not the full Axios response
- Generic types on `api.get<T>()` / `api.post<T>()` for type safety
- Pagination follows `{ data: T[], meta: { total, page, lastPage } }` shape

### Backend NestJS Patterns
- **Module structure**: Each domain has `module.ts`, `controller.ts`, `service.ts`, `dto/` folder
- **Dependency injection**: Services injected via constructor (`private readonly prisma: PrismaService`)
- **Error handling**: NestJS built-in exceptions (`NotFoundException`, `BadRequestException`)
- **Transactions**: `prisma.$transaction()` for multi-step writes (e.g., voting)
- **DTOs**: Separate DTO classes for input validation with class-validator decorators
- **Pagination**: Consistent `{ data, meta: { total, page, lastPage } }` response shape

### Backend Service Pattern
```typescript
@Injectable()
export class QuestionsService {
  constructor(private readonly prisma: PrismaService) {}

  async findAll(certificationId?: string, page = 1, limit = 10) {
    const [total, items] = await Promise.all([
      this.prisma.model.count({ where }),
      this.prisma.model.findMany({ where, include: {...}, orderBy, skip, take }),
    ]);
    return { data: items, meta: { total, page, lastPage: Math.ceil(total / limit) } };
  }
}
```

## UI/Styling Patterns

### Component Composition (shadcn-ui)
- All UI primitives in `src/components/ui/` — generated by shadcn-ui CLI, built on Radix UI
- Components use `React.forwardRef` with `cn()` utility for className merging
- Variant system via `class-variance-authority` (cva):
```tsx
const buttonVariants = cva("base-classes", {
  variants: { variant: { default: "...", outline: "..." }, size: { default: "...", sm: "..." } },
  defaultVariants: { variant: "default", size: "default" },
});
```
- `asChild` prop pattern (via Radix `Slot`) for polymorphic rendering

### Tailwind CSS Patterns
- Dark theme by default with CSS custom properties (`bg-background`, `text-foreground`, `text-muted-foreground`)
- Glass morphism cards: `glass-card` custom class
- Glow effects: `glow-cyan`, `glow-green` custom classes
- Gradient text: `text-gradient-cyan`, `text-gradient-warm` custom classes
- Grid background: `bg-grid` custom class
- Responsive: `md:` and `lg:` breakpoints for layout changes
- Spacing: consistent use of Tailwind spacing scale (`gap-2`, `p-6`, `mb-4`)
- Font: `font-mono` for headings, labels, and technical text

### Animation Patterns (Framer Motion)
- Page-level entrance animations with `motion.div`:
```tsx
<motion.div initial={{ opacity: 0, y: 20 }} animate={{ opacity: 1, y: 0 }} transition={{ duration: 0.7 }}>
```
- Scroll-triggered animations: `whileInView` with `viewport={{ once: true }}`
- Staggered list animations: `transition={{ delay: i * 0.1 }}`
- `AnimatePresence` for enter/exit transitions on dynamic lists

### Form Patterns
- Controlled inputs with individual `useState` per field (not react-hook-form in page components)
- Toast notifications via `useToast()` for validation errors and success messages
- Form validation inline in `handleSubmit` before API call
- Vietnamese language for user-facing toast messages and labels

## Routing & Auth Patterns
- Routes defined in `App.tsx` using react-router-dom v6 `<Routes>` / `<Route>`
- Protected routes wrap children with `<ProtectedRoute>` component
- Auth guard checks `useAuthStore.isAuthenticated`, redirects to `/auth` with location state
- JWT tokens stored in Zustand persist store (localStorage)
- Automatic token refresh via Axios response interceptor on 401

## Data Patterns
- Mock data in `src/data/mockData.ts` for development/fallback
- Type definitions in `src/types/exam.ts` shared across components and services
- IDs are string UUIDs throughout
- Choices use single-char labels (`'a'`, `'b'`, `'c'`, `'d'`)
